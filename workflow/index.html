1. Een blog-workflow

Inleiding

Eén van de belangrijkste stappen is nu voldaan, de blog die gebruikt zal worden voor mijn GIP staat online en werkt.

De blog die u nu leest maakt gebruik het blog framework Hexo, andere mooie weetjes zijn:

  • Hexo maakt gebruik van Node.js voor het generen van de pagina’s, dit zou ervoor moeten zorgen dat een grote hoeveelheid van pagina’s maar enkele seconden nodig hebben om gegenereerd te worden.
  • Markdown support: Hexo ondersteunt alle mogelijkheden dat de welkgekende Github Flavored Markdown ook kan, zelfs plugins speciaal voor Octopress. Deze pagina is zelf gemaakt in Github Flavored Markdown.

De installatiefase

Zoals alle frameworks die men gebruikt voor blogging en websites moet je deze eerst installeren op je eigen computer, Hexo heeft in totaal 5 commando’s die uitgevoerd moeten worden en je blog te kunnen uitvoeren (weliswaar met enkel een ‘Hello World’ pagina).

Eerst en vooral moet Node.js en Git geïnstalleerd worden op de computer, Hexo heeft beiden nodig om deze te installeren en uiteindelijk de website te kunnen genereren en uitvoeren.

Na de installatie van beide programma’s open je een command prompt (cmd-prompt) en geef je de volgende commando’s in.

$ npm install hexo -g

Met dit commando vertel je npm dat deze Hexo moet installeren op een ‘global’ niveau.

$ Hexo init blog

Dit commando kan je uitvoeren als Hexo succesvol door npm is geïnstalleerd, dit maakt een folder genaamd blog aan in het venster waar de cmd-prompt geopend was (in dit geval was dat C:\Users\Administrator).

$ cd blog

Een cmd-prompt commando om de folder blog te openen.

$ npm install

Dit commando moet je runnen omdat Hexo alle component van npm nodig heeft in de map om te kunnen functioneren.

$ hexo server

Dit commando moet je uitvoeren als je je server wilt kunnen bekijken in zijn ruwste vorm, een bericht dat je naar http://localhost:4000 verschijnt dan. Hier kan je dan een ‘Hello World’ pagina vinden die je meer uitleg geeft over hoe je Hexo kan gebruiken.

De vormgevingsfase

Website Informatie

Na het installeren en onze ‘blog’ voor de eerste keer eventjes bekeken te hebben kom je er snel achter dat men deze moet personaliseren.

Eerst en vooral heb ik de blog map geopend en hier even door gekeken, hier opende ik _config.yml, dit bestand bevat alle configuraties van je ‘blog’. Hier heb ik wat basis informatie ingevuld voor de site:

#Site
title: GIP - Alexander Maes - 6IB
subtitle: Immaculata Instituut Oostmalle
description: Geïntegreerde Proef voor het 6de Middelbaar TSO Informatica-Beheer
author: Alexander Maes
email: alexander.maes@immalle.be
language: Dutch

Onze eerste pagina

Hierna heb ik een cmd-prompt venster geopend en heb ik volgend commando ingegeven:

$ hexo new page "workflow"

Dit was om een nieuwe pagina te maken, deze in de URL is dan ook te zien als /workflow/. Om dit in de header te krijgen moest ik wel naar themes en dan landscape gaan. In deze map zit ook een _config.yml bestand en dit is specifiek voor het thema dat onze Hexo op dit moment gebruikt. Ik heb hier de eerste paar lijnen aangepast.

# Header
menu:
    0. ´Hello World´: /
    1. Een blog-workflow: /workflow
rss: false

Hier heb ik dan de titel van het header-item ingegeven 1. Een blog-workflow en heb ik hier /workflow ingevuld, dezelfde naam als de map in mijn source map (en tevens ook de naam van het new page commando). Hiernaast heb ik ook rss op false gezet omdat we hier waarschijnlijk geen gebruik van maken.

Geen blog, maar een website

Een beetje verder in dit document vinden we dan volgende lijnen:

# Sidebar
sidebar: false

Hier heb ik sidebar op false gezet omdat we Hexo gaan gebruiken niet voor zijn blog-capaciteiten, maar vooral om er een gewone website van te maken. De sidebar die Hexo gebruikt is zeer kenmerkend voor blogs en hebben we dus niet nodig.

Nu hebben we onze nieuwe (lege) pagina genaamd 1. Een blog-workflow en een item in de header die ons hiernaar verwijst. Nu gaan we de website nog wat personaliseren door zelf een banner te kiezen en ook zetten we nog tekst op deze nieuwe workflow pagina.

Onze banner

Bovenaan onze pagina heb je een afbeelding die men ook wel onze banner noemt, we gaan deze vervangen met iets persoonlijkers en niet de standaardafbeelding die Hexo gebruikt.

Van onze landscape folder gaan we verder door naar de source folder, dan css en uiteindelijk images, hier zien we één bestand in staan en dit heet banner.jpg. We kunnen nu twee manieren volgen om een nieuwe banner te zetten:

  1. We verwijderen banner.jpg en zoeken een andere afbeelding (1900x1200) die we dan hernoemen naar banner.jpg en deze dan in deze map zetten.
  2. We zoeken een andere afbeelding en droppen deze in deze map onder een andere naam, bv. mijn_banner.jpg. Hierna gaan we een map terug zodat we in de css map zitten. Hier vinden we een bestand genaamd _variables.styl en dit openen we dan. Zoek naar het de lijn banner-url = "images/banner.jpg" en verander het banner.jpg gedeelte naar mijn_banner.jpg zodat we banner-url = "images/banner2.jpg" krijgen en slaag deze dan op.

Na één van deze twee manieren te gebruiken zal de banner zijn verandert, dit gebeurt automatisch en heeft geen hexo server commando nodig.

Tekst

Nadat we alle vorige stappen hebben gedaan en onze website een nieuwe (mooie) banner gegeven te hebben kunnen we nu tekst in onze workflow pagina zetten. We gaan hiervoor terug naar de blog map en openen source en hierna workflow, elke nieuwe pagina zal hier zijn eigen mapje krijgen en de _posts map is voor alle posts (blog-gerelateerd) op de hoofdpagina.

In de workflow map zal één bestandje staan dat index.md heet, we gaan dit openen in Notepad++ en we krijgen de volgende output:

title: workflow
date: 2014-10-08 15:25:21
---

Je kan hier title veranderen naar bv. 1. Een blog-workflow in plaats van workflow, de date werd automatisch gegeneerd toen we deze pagina gemaakt hadden. De drie lijntjes signaleren dat het informatie-gedeelte van de pagina voorbij is en na deze lijntjes kan je door gebruik te maken van (Github Flavored) Markdown of HTML gewoon een tekst schrijven. De eerste paar lijnen van deze pagina zijn het volgende:

title: 1. Een blog-workflow
date: 2014-10-08 15:25:21
---

Een eindwoord

Zodus hebben we in de bovenstaande stappen snel getoond hoe men Hexo moet installeren, de website informatie kan ingeven, een eerste pagina moet aanmaken, hoe je een gedeelte van de opmaak wijzigt zoals de side-bar en de banner en als laatste hoe je tekst ingeeft op een nieuwe pagina (of post) in Markdown.
